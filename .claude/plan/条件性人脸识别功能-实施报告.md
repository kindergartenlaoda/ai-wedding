# 条件性人脸识别功能 - 实施报告

## 实施时间
2026-02-26

## 功能概述
根据 domain 的 `require_face_detection` 配置，动态决定是否对上传的照片进行人脸识别验证。

---

## 代码修改清单

### 1. `app/hooks/useParallelPhotoUpload.ts`

#### 修改内容
1. **接口扩展**：添加 `requireFaceDetection: boolean` 参数
2. **API 还原**：将 `identifyPerson` 从 Mock 还原为真实 API 调用
3. **逻辑分流**：
   - `requireFaceDetection = false`：直接标记为 `valid`，跳过识别
   - `requireFaceDetection = true`：调用 `/api/identify-image` 进行识别
4. **错误语义修复**：区分服务失败（`error`）和无人像（`invalid`）
5. **未登录保护**：需要识别时，未登录用户上传会提前拦截

#### 关键代码
```typescript
// 如果不需要人脸识别，直接标记为 valid
if (!requireFaceDetection) {
  dispatch({
    type: 'UPDATE_PHOTO',
    photoId: photo.id,
    updates: { identifyStatus: 'valid' },
  });
  return;
}

// 需要人脸识别，执行真实 API 调用
const result = await identifyPerson(photo.dataUrl);

// 区分服务失败和无人像
if (result && !result.success) {
  throw new Error(result.description || '识别服务失败');
}
```

const handleFileChange = useCallback(
  (e: React.ChangeEvent<HTMLInputElement>) => {
    if (e.target.files) {
      // 如果需要人脸识别且未登录，提示先登录
      if (requireFaceDetection && !user) {
        setShowAuthModal(true);
        e.target.value = '';
        return;
      }
      handleFiles(Array.from(e.target.files));
      e.target.value = '';
    }
  },
  [handleFiles, requireFaceDetection, user]
);

## 问题修复记录

### 代码审查发现的问题及修复

#### 1. P0 - 规则绕过风险
**问题**：`require_face_detection` 仅在前端控制，且默认为 `false`（fail-open）

**修复状态**：部分修复
- ✅ 前端逻辑已实现条件判断
- ⚠️ 后端强制校验需要在后续 PR 中添加（修改生成接口）

**后续 TODO**：
- 在 `/api/generate-image` 和 `/api/generate-stream` 中添加后端验证
- 检查 template/domain 的 `require_face_detection` 配置
- 如果要求识别但照片未通过验证，拒绝生成请求

---

#### 2. P1 - 未登录用户流程冲突
**问题**：识别接口需要登录，但用户在上传阶段可能未登录

**修复状态**：✅ 已修复
- 在 `handleFileChange` 和 `handleDrop` 中添加前置登录检查
- 需要识别时，未登录用户上传会立即显示登录弹窗
- 避免了"上传后识别失败 → 登录 → 无法重试"的困境

---

#### 3. P1 - 识别失败语义混淆
**问题**：服务失败被当作"无人像"（`invalid`）

**修复状态**：✅ 已修复
- 在 `identifyPerson` 中检查 `result.success` 字段
- 服务失败时抛出错误，映射为 `identifyStatus='error'`
- 无人像时返回 `hasPerson=false`，映射为 `identifyStatus='invalid'`
- 用户可以清晰区分"服务异常"和"未检测到人物"

1. 选择"AI 风景壁纸"（`require_face_detection = false`）
2. 上传任意照片 → 预期：直接标记为 `valid`，无识别过程
3. 未登录也可以上传 → 预期：正常上传，不触发登录弹窗
