# 条件性人脸识别功能 - 完整实施报告

## 实施时间
2026-02-26

## 功能概述
根据 domain 的 `require_face_detection` 配置，动态决定是否对上传的照片进行人脸识别验证。包含前端逻辑分流和后端强制校验两层防护。

  (e: React.ChangeEvent<HTMLInputElement>) => {
    if (e.target.files) {
      // 如果需要人脸识别且未登录，提示先登录
      if (requireFaceDetection && !user) {
        setShowAuthModal(true);
        e.target.value = '';
        return;
      }
      handleFiles(Array.from(e.target.files));

### 4. `app/api/generations/route.ts` ✅ (新增)

#### 修改内容
1. **查询 template 和 domain 配置**：在创建 generation 时查询 template 和 domain 信息
2. **后端强制校验**：检查 `require_face_detection` 配置，验证照片是否已上传
3. **fail-closed 策略**：如果要求识别但照片为空，拒绝创建 generation

#### 关键代码
```typescript
// 后端强制校验：如果 domain 要求人脸识别，验证照片是否已通过识别
if (generation_type === 'batch' && template && project) {
  const domainSlug = template.domain || domain || 'wedding';
  const domainConfig = await prisma.domains.findUnique({
    where: { slug: domainSlug },
    select: { require_face_detection: true },
  });

  if (domainConfig?.require_face_detection) {
    const uploadedPhotos = project.uploaded_photos as string[] | null;
    if (!uploadedPhotos || uploadedPhotos.length === 0) {
      return NextResponse.json(
        { error: '该领域要求上传照片并通过人脸识别验证' },
        { status: 400 }
      );
    }
  }
}
```

## 问题修复记录

### 代码审查发现的问题及修复

#### 1. P0 - 规则绕过风险 ✅ 已完全修复
**问题**：`require_face_detection` 仅在前端控制，且默认为 `false`（fail-open）

**修复状态**：✅ 已完全修复
- ✅ 前端逻辑已实现条件判断
- ✅ 后端强制校验已添加（`/api/generations` 接口）
- ✅ 采用 fail-closed 策略：要求识别但照片为空时拒绝请求

**修复效果**：
- 即使前端被绕过，后端也会拒绝不符合要求的生成请求
- 双层防护确保业务规则不被违反

---

#### 2. P1 - 未登录用户流程冲突 ✅ 已修复
**问题**：识别接口需要登录，但用户在上传阶段可能未登录

**修复状态**：✅ 已修复
- 在 `handleFileChange` 和 `handleDrop` 中添加前置登录检查
- 需要识别时，未登录用户上传会立即显示登录弹窗
- 避免了"上传后识别失败 → 登录 → 无法重试"的困境

**修复效果**：
- 用户体验流畅，不会出现识别失败后无法继续的情况
- 登录后可以正常上传和识别照片

---

#### 3. P1 - 识别失败语义混淆 ✅ 已修复
**问题**：服务失败被当作"无人像"（`invalid`）

**修复状态**：✅ 已修复
- 在 `identifyPerson` 中检查 `result.success` 字段
- 服务失败时抛出错误，映射为 `identifyStatus='error'`
- 无人像时返回 `hasPerson=false`，映射为 `identifyStatus='invalid'`

**修复效果**：
- 用户可以清晰区分"识别服务异常"和"未检测到人物"
- 错误提示更准确，不会误导用户操作
