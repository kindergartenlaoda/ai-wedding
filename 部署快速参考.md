## ⚡ 一行命令部署

### Windows
```powershell
.\deploy.ps1
```

### macOS / Linux
```bash
bash deploy.sh
```

**首次运行会提示输入**:
1. AI API Key（必填）
2. 管理员邮箱（默认: admin@example.com）
3. 管理员密码（默认: 123456）

## 🔧 快速修改配置

### 修改端口
编辑 `.env` 文件：
```bash
APP_PORT=8080           # 应用端口（默认 3000）
POSTGRES_PORT=5434      # 数据库端口（默认 5433）
MINIO_API_PORT=9002     # MinIO API（默认 9000）
```

### 切换 AI 提供商
编辑 `.env` 文件：
```bash
# OpenAI
IMAGE_API_BASE_URL=https://api.openai.com
IMAGE_API_KEY=sk-xxx

# 302.ai
IMAGE_API_BASE_URL=https://api.302.ai/v1
IMAGE_API_KEY=sk-xxx

# OpenRouter
IMAGE_API_BASE_URL=https://openrouter.ai/api/v1
IMAGE_API_KEY=sk-xxx
```

修改后重启：
```bash
.\deploy.ps1 restart      # Windows
bash deploy.sh restart    # macOS/Linux
```

## ⚠️ 常见问题快速解决

### Windows: "禁止运行脚本"
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

### Docker Desktop 未启动
手动启动 Docker Desktop，等待就绪后重新运行脚本

### 端口被占用
```bash
# 查看占用（Windows）
Get-NetTCPConnection -LocalPort 3000

# 查看占用（Linux/macOS）
lsof -i :3000

# 或修改 .env 中的端口配置
```

### MinIO 403 错误
```bash
# Docker 环境
docker compose exec app pnpm fix-minio

# PM2 环境
pnpm fix-minio
```

### 完全重置
```bash
# 停止并删除所有数据
docker compose down -v

# 清理 Docker 缓存
docker system prune -af

# 重新部署
.\deploy.ps1              # Windows
bash deploy.sh            # macOS/Linux
```
